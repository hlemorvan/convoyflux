== Prérequis

* https://docs.docker.com/get-docker/[Docker] ≥ 24 avec Docker Compose v2
* Ports libres : *3000*, *8080*, *7000*, *1883*, *5432*

== Lancer l'application

[source,bash]
----
git clone https://github.com/hlemorvan/convoyflux.git
cd convoyflux
cp .env.example .env        # optionnel — les valeurs par défaut fonctionnent
docker compose up --build
----

NOTE: Le premier démarrage télécharge les images Docker et compile le backend Java (~3 min).
Les suivants utilisent le cache Docker et démarrent en quelques secondes.

== Ouvrir l'interface

Une fois les conteneurs démarrés, ouvrir : *http://localhost:3000*

....

      │                     │                    │                    │                │
      │ ouvre :3000         │                    │                    │                │
      │────────────────────►│                    │                    │                │

      │   jour sur carte    │                    │                    │                │
....

== Vérifier que tout tourne

[source,bash]
----
# État des conteneurs
docker compose ps

# Flux SSE (Ctrl+C pour arrêter)
curl http://localhost:8080/api/stream

# Santé du backend
curl http://localhost:8080/actuator/health
----

Réponse attendue de l'actuator :

[source,json]
----
{"status":"UP","groups":["liveness","readiness"]}
----

== Arrêter l'application

[source,bash]
----
docker compose down          # arrête et supprime les conteneurs
docker compose down -v       # idem + supprime le volume PostgreSQL
----

WARNING: `docker compose down -v` supprime définitivement les données PostgreSQL.
