== Modèle du domaine

Le module `domain` est en *Java pur* — aucune dépendance Spring, uniquement
`reactor-core` pour `Flux`/`Mono` dans les interfaces de port.

....

== Value Objects

Tous les modèles du domaine sont des *Java records* avec validation dans le
constructeur compact :

[source,java]
----
public record VehicleId(String value) {
    public VehicleId {
        if (value == null || value.isBlank())
            throw new IllegalArgumentException("VehicleId must not be blank");
    }
}

public record GeoPoint(double lat, double lng) {
    public GeoPoint {
        if (lat < -90  || lat > 90)   throw new IllegalArgumentException("lat out of range");
        if (lng < -180 || lng > 180)  throw new IllegalArgumentException("lng out of range");
    }
}
----

== Règle de backpressure

`TelemetrySinkBroadcast` utilise `Sinks.many().multicast().directBestEffort()` :
si un abonné SSE est lent, ses trames sont droppées *individuellement* sans
bloquer les autres ni le producteur MQTT.

[source,java]
----
// directBestEffort() : multicast sans blocage, drop individuel pour les lents
// (onBackpressureLatest n'existe que sur unicast — ne pas utiliser ici)
private final Sinks.Many<Telemetry> sink =
    Sinks.many().multicast().directBestEffort();
----
